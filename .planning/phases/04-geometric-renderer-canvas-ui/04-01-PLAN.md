    ## Render Types (src/lib/render/types.ts)
    Define all types consumed by the renderer:

    - `ShapeType = 'rect' | 'circle' | 'triangle' | 'line' | 'empty'`
    - `Cell { x, y, width, height, depth }` — rectangular region from subdivision
    - `SceneElement { type: ShapeType, x, y, width, height, fill: string, stroke?: string, strokeWidth?: number, opacity: number, area: number, depth: number }` — single drawing instruction
    - `SceneGraph { elements: SceneElement[], width: number, height: number, background: string }` — complete scene
    - `RenderConfig { complexity, symmetry, density, curvature, texture, scaleVariation, rhythm, regularity, directionality, layering, energy, minCellSize: 4, cellGap: number (2-4px), framePadding: number }` — derived from ParameterVector

    ## Subdivision Tests (src/__tests__/render/subdivide.test.ts)
    - GEOM-01: subdivide(region, config, prng) produces an array of Cell objects that tile the input region
    - GEOM-02 min-size: At complexity=1.0, no returned cell has width or height less than 4px
    - GEOM-02 depth-limiting: At complexity=1.0 + density=1.0, verify cell count is bounded (not infinite recursion)
    - GEOM-03 frame-padding: Root region starts inset by framePadding (2% of canvasSize = 16px for 800x800), all cells within this boundary
    - Subdivision depth: complexity=0 produces fewer cells than complexity=1 (depth scales with complexity)
    - Symmetry influence: symmetry=1.0 produces more balanced splits (ratio closer to 0.5) vs symmetry=0.0
    - Cell gap: Adjacent cells separated by cellGap (2-4px) — no overlap between returned cells

    ## Shape Assignment Tests (within subdivide.test.ts)
    - curvature=1.0: majority of shapes are circles
    - curvature=0.0: majority of shapes are rectangles/triangles (few circles)
    - density parameter: low density leaves more cells as 'empty' type
    - All non-empty shapes assigned a fill color from the palette

    **Step 3 — Implement src/lib/render/geometric/subdivide.ts (GREEN):**

    `subdivide(region: Cell, config: RenderConfig, prng: () => number, depth?: number): Cell[]`

    Algorithm per CONTEXT.md locked decisions:
    - Max depth = 2 + floor(complexity * 6), range 2-8
    - Terminal conditions: depth >= maxDepth, or width < minCellSize * 2 + cellGap, or height < minCellSize * 2 + cellGap
    - Split direction: bias toward longer side (aspectBias = width > height ? 0.7 : 0.3). directionality modulates this bias (high = stronger preference for one direction). rhythm controls whether splits alternate predictably (high = more alternation).
    - Split ratio: symmetry pulls toward 0.5. Range = [0.5 - (1-symmetry)*0.3, 0.5 + (1-symmetry)*0.3]. regularity at high values snaps ratio to clean fractions (1/2, 1/3, 2/3).
    - scaleVariation widens allowed ratio range (high = more asymmetric splits allowed)
    - Child regions: two sub-regions with cellGap between them
    - Frame padding: root call starts with region inset by framePadding on all sides (GEOM-03)

    **Step 4 — Implement src/lib/render/geometric/shapes.ts (GREEN):**

    `assignShape(cell: Cell, config: RenderConfig, palette: PaletteColor[], prng: () => number): SceneElement`

    Per CONTEXT.md locked decisions:
    - curvature drives shape distribution: P(circle) = curvature * 0.5, P(triangle) = (1-curvature) * 0.3, P(rect) = 1 - P(circle) - P(triangle), small P(line) always ~0.05
    - density controls fill vs empty: fillProbability = 0.3 + density * 0.65, range 0.3-0.95
    - If prng() > fillProbability, return 'empty' element
    - Color assignment: first palette color is dominant (60% probability for largest cells by area), remaining distributed among other palette colors. energy parameter increases diversity of color usage (high energy = more palette colors used).
    - layering: at high layering (>0.7), some shapes extend slightly beyond cell bounds (add 5-15% overflow)
    - Stroke: if config.strokeVisible, add stroke using contrasting palette color or neutral. Use config.primaryStrokeWidth for large shapes (area > median), config.secondaryStrokeWidth for small shapes (GEOM-04: at most 2 weights).

    **Step 3 — Implement src/lib/render/geometric/scene.ts (GREEN):**

    `buildSceneGraph(params: ParameterVector, palette: PaletteResult, theme: 'dark' | 'light', seed: string, canvasSize?: number): SceneGraph`

    Implementation:
    1. Create PRNG from seed via `createPRNG(seed)`
    2. Derive RenderConfig from ParameterVector (map each param to config values per Task 1 types)
    3. Compute framePadding = canvasSize * 0.02 (default canvasSize = 800)
    4. Create root Cell inset by framePadding on all sides
    5. Call `subdivide(rootCell, config, prng)` to get cells
    6. Select palette colors based on theme: `theme === 'dark' ? palette.dark : palette.light`
    7. For each cell, call `assignShape(cell, config, paletteColors, prng)` to get SceneElement
    8. Sort elements by area descending (largest first for progressive animation)
    9. Set background: dark theme → first palette dark color with very low lightness, or use css variable value. Use palette background approach: for dark mode use a near-black hex (#0a0a0a or similar), for light mode use near-white (#fafafa or similar)
    10. Return SceneGraph

    **Step 4 — Implement src/lib/render/geometric/draw.ts (GREEN):**

    `drawElement(ctx: CanvasRenderingContext2D, el: SceneElement, alpha?: number): void`
    `drawSceneComplete(ctx: CanvasRenderingContext2D, scene: SceneGraph): void`

    drawElement:
    - Set ctx.globalAlpha = alpha (default 1.0)
    - Skip 'empty' type elements
    - Switch on el.type: rect → fillRect, circle → arc + fill, triangle → moveTo/lineTo path + fill, line → moveTo/lineTo + stroke
    - If el.stroke && el.strokeWidth → overlay stroke on shape
    - Reset ctx.globalAlpha = 1.0 after drawing

    drawSceneComplete:
    - Clear canvas: ctx.fillStyle = scene.background, fillRect(0, 0, scene.width, scene.height)
    - Iterate all elements, call drawElement(ctx, el, 1.0)
