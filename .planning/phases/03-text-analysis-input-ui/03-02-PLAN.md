  <behavior>
    - useTextAnalysis returns { stage, result, generate, reset }
    - stage is 'idle' | 'parsing' | 'analyzing' | 'normalizing' | 'rendering' | 'complete'
    - Calling generate(text) runs the full pipeline: canonicalize -> analyze -> calibrate+normalize+map -> palette
    - Each stage has minimum 200ms display time (smoothing) so fast steps don't flash by
    - result is null when idle/generating, populated when complete
    - result contains: { vector, provenance, palette, summaries, canonical, changes }
    - reset() returns to idle state with null result
    - Calibration distributions are computed once and cached (not recomputed per generation)
    - PipelineProgress component shows 4 stages: Parsing, Analyzing, Normalizing, Rendering
    - Active stage has violet color (var(--color-accent)), completed stages show checkmark and dim, future stages are muted
    - Component has role="progressbar" and aria-label for accessibility
    - Respects prefers-reduced-motion: if reduced motion preferred, skip stage animation delays
  </behavior>
  <action>
    Create `src/hooks/useTextAnalysis.ts` as a 'use client' hook:
    - Type PipelineStage = 'idle' | 'parsing' | 'analyzing' | 'normalizing' | 'rendering' | 'complete'
    - Type PipelineResult = { vector: ParameterVector; provenance: ParameterProvenance[]; palette: PaletteResult; summaries: Record<string, string>; canonical: string; changes: string[] }
    - Cache calibration data in a module-level variable (compute once on first generate, reuse after)
    - The generate function:
      1. setStage('parsing') -> await minDelay(200) -> canonicalizeText(text)
      2. setStage('analyzing') -> await minDelay(200) -> analyzeText(canonical)
      3. setStage('normalizing') -> await minDelay(200) -> get cached calibration -> computeParameterVector -> generateAllSummaries
      4. setStage('rendering') -> await minDelay(200) -> sha256(canonical) for seed -> generatePalette(vector, seed)
      5. setResult({ vector, provenance, palette, summaries, canonical, changes }) -> setStage('complete')
    - minDelay checks window.matchMedia('(prefers-reduced-motion: reduce)').matches — if true, skip delay
    - reset() sets stage to 'idle' and result to null

    Create `src/hooks/usePipelineProgress.ts` — optional thin wrapper if needed, or inline in PipelineProgress.

    Create `src/components/progress/PipelineProgress.tsx`:
    - 'use client' component
    - Props: { currentStage: PipelineStage }
    - Render horizontal row of 4 stage labels: Parsing, Analyzing, Normalizing, Rendering
    - Each label shows: checkmark icon (completed), violet highlight (active), muted text (pending)
    - Use font-mono for stage labels
    - Wrap in role="progressbar" aria-label="Generation progress" aria-valuenow (stage index) aria-valuemax (4)
    - Smooth transition on color changes (transition-colors)
